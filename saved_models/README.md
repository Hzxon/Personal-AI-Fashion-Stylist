# saved_models — Artifact Directory

This directory contains model checkpoints and JSON artifacts produced by the O4U ML pipeline.

---

## Model Checkpoints (`.pth` files)

### `best_hybrid_model.pth` ✅ PRIMARY

| Field | Value |
|---|---|
| **Origin script** | `scripts/train.py` |
| **Architecture** | `HybridFashionModel` — `VisualBranch` (Transformer + attention pooling) fused with a physical feature branch via `FiLMFusion` (concat → MLP with LayerNorm + Dropout) |
| **Checkpoint criterion** | Best validation ROC-AUC (configurable via `--save-metric`) |
| **Threshold** | Classification threshold from `thresholds.json` (Youden's J on calibration split); current value: **0.47** |
| **Calibration ROC-AUC** | 0.82 (from `thresholds.json`) |

This is the canonical model checkpoint. Use this file for inference, evaluation, and ensemble feature extraction.

---

### `best_model.pth` ⚠️ DEPRECATED

> **This file is deprecated. Use `best_hybrid_model.pth` instead.**

`best_model.pth` cannot be attributed to a specific training run. Its origin script, architecture version, and validation metrics are unknown. It predates the hardened pipeline introduced in the `pipeline-hardening` spec and may correspond to an earlier, undocumented architecture (possibly using `GatedFusion` instead of `FiLMFusion`, or `BatchNorm1d` instead of `LayerNorm`).

Do not use this file for inference or evaluation. It is retained only to avoid breaking any external references that may point to it.

## JSON Artifacts

Artifacts are grouped by when they are generated.

### Generated at training time (`scripts/train.py`)

| File | Purpose |
|---|---|
| `phys_feature_cols.json` | Ordered list of physical feature column names used as model input, including `_was_imputed` indicator columns. Loaded by `scripts/inference.py` and `scripts/evaluate_test_set.py` to reconstruct the exact feature vector. |
| `encoder_mapping.json` | Maps each categorical column to its one-hot encoded column names, and lists `body_figure` classes. Derived from `concat(train, val)` before splitting to ensure all categories are represented. Loaded at inference time instead of re-deriving from the training manifest. |
| `training_config.json` | Model card: architecture hyperparameters, training hyperparameters (LR, batch size, epochs, seed), dataset split sizes, feature set mode (`full` or `ablated`), binary label threshold, and ISO timestamp of the training run. |
| `thresholds.json` | Optimal classification threshold selected by maximizing Youden's J (`sensitivity + specificity − 1`) on the held-out calibration split (20% of training data). Includes the method used and calibration ROC-AUC. |
| `score_normalization.json` | Mean and standard deviation of `score` on the deep-train split, used to z-score normalize regression targets during training. Loaded at inference time to inverse-transform predicted regression outputs. *(Generated when `scripts/train.py` completes a full run.)* |
| `artifact_manifest.json` | Lists all artifact filenames, their purposes, and the training run timestamp. Provides a single source of truth for auditing what was saved in a given training run. *(Generated when `scripts/train.py` completes a full run.)* |
| `ablation_results.json` | ROC-AUC and F1 on the validation split for both `ablated` (6 EDA-flagged traits excluded) and `full` feature set modes. Generated when `scripts/train.py` is run with `--feature-set ablated`. *(Generated on ablation run.)* |

### Generated at evaluation time (`scripts/evaluate_test_set.py`, `scripts/baselines.py`, `scripts/threshold_sensitivity.py`)

| File | Purpose |
|---|---|
| `test_evaluation_results.json` | Full evaluation results on the held-out test split: ROC-AUC, F1, accuracy, Brier score, MAE, 95% bootstrap confidence intervals (1,000 resamples), and uncertainty estimates when multiple imputation is used. *(Generated by `scripts/evaluate_test_set.py`.)* |
| `per_group_metrics.json` | Accuracy and F1 broken down by `body_figure` group and by imputed-vs-observed rows. Identifies subgroup performance disparities. *(Generated by `scripts/evaluate_test_set.py`.)* |
| `baseline_results.json` | ROC-AUC and F1 for three baselines on the validation split: pure-visual (logistic regression on CLIP features), pure-physical (logistic regression on imputed physical features), and CLIP zero-shot cosine similarity. *(Generated by `scripts/baselines.py`.)* |
| `threshold_sensitivity.json` | Sensitivity analysis of the binary label threshold used to convert regression scores to binary labels. Evaluates thresholds `[0.5, 1.0, 1.5, 2.0]` and reports class balance (positive rate) and downstream model ROC-AUC for each. *(Generated by `scripts/threshold_sensitivity.py`.)* |

### Generated at notebook time (NB05, NB06, NB07)

| File | Purpose |
|---|---|
| `nb05_best_params.json` | Best Optuna hyperparameters found by `05_Bagging_Ensemble.ipynb` for `RandomForestClassifier`. *(Generated by running NB05.)* |
| `nb06_best_params.json` | Best Optuna hyperparameters found by `06_Stacking_Ensemble.ipynb` for the stacking ensemble. *(Generated by running NB06.)* |
| `nb07_best_params.json` | Best Optuna hyperparameters found by `07_Custom_Stacking_Ensemble.ipynb` for `XGBClassifier`. *(Generated by running NB07.)* |

## Regenerating Artifacts

To regenerate all training-time artifacts from scratch:

```bash
# 1. Run the imputation pipeline
python3 scripts/o4u_imputation_pipeline.py

# 2. Run training (produces best_hybrid_model.pth and all training-time JSONs)
python3 scripts/train.py --seed 42 --save-metric roc_auc --feature-set full

# 3. Run test evaluation (produces test_evaluation_results.json, per_group_metrics.json)
python3 scripts/evaluate_test_set.py

# 4. Run baselines (produces baseline_results.json)
python3 scripts/baselines.py

# 5. Run threshold sensitivity analysis (produces threshold_sensitivity.json)
python3 scripts/threshold_sensitivity.py
```

Notebook artifacts (`nb05_best_params.json`, `nb06_best_params.json`, `nb07_best_params.json`) are generated by running the corresponding Jupyter notebooks in `notebooks/`.
